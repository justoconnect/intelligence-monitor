# Contributing

Merci de l'interet porte au projet. Les contributions — bugs, features, docs — sont les bienvenues.

## Workflow

1. **Fork** le repo et creez une branche feature : `git checkout -b feat/ma-feature`.
2. Ecrivez du code clair, avec **type hints** et **docstrings**.
3. Lancez le linter localement : `python -m py_compile intelligence_monitor.py`.
4. **Commit** avec un message descriptif (Conventional Commits recommande).
5. Ouvrez une **Pull Request** vers `main` et remplissez le template.

## Style

- Python 3.9+.
- Utilisez la stdlib autant que possible — ce projet reste **zero dependances**.
- Noms explicites en anglais dans le code, docs en FR/EN selon contexte.
- Pas de secrets commites — utilisez `.env` ignore.

## Issues

Utilisez les templates **Bug report** ou **Feature request** dans `.github/ISSUE_TEMPLATE`.

## Code of conduct

Respect, bienveillance, focus sur le probleme. Pas de harcelement tolere.
